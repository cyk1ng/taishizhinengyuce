## 1. Oracle数据库接入

### 1.1 修改数据库配置

#### 方式一：使用环境变量（推荐）

在项目根目录创建 `.env` 文件：

```bash
# 数据库类型配置
DB_TYPE=oracle

# Oracle数据库连接信息
DB_HOST=your_oracle_host          # Oracle服务器地址
DB_PORT=1521                       # Oracle端口，默认1521
DB_NAME=ORCL                       # Oracle服务名（SID）
DB_USER=your_username              # 数据库用户名
DB_PASSWORD=your_password          # 数据库密码

# 或者使用完整的DATABASE_URL（优先级更高）
# DATABASE_URL=oracle+cx_oracle://username:password@host:1521/?service_name=ORCL
```

#### 方式二：直接修改代码

修改 `src/storage/database/db.py` 第61-69行：

```python
elif db_type == "oracle":
    host = os.getenv("DB_HOST", "your_oracle_host")       # 修改为你的Oracle服务器地址
    port = os.getenv("DB_PORT", "1521")                   # 确认端口
    name = os.getenv("DB_NAME", "ORCL")                   # 确认服务名
    user = os.getenv("DB_USER", "your_username")          # 修改为你的数据库用户名
    password = os.getenv("DB_PASSWORD", "your_password")  # 修改为你的数据库密码
    # Oracle URL 格式: oracle+cx_oracle://user:password@host:port/?service_name=name
    url = f"oracle+cx_oracle://{user}:{password}@{host}:{port}/?service_name={name}"
    return url
```

### 1.3 检查表结构兼容性

Oracle与MySQL在SQL语法上有一些差异，需要检查以下文件中的SQL语句：

#### `assets/sql/workload_tables.sql`

如果需要将MySQL语法转换为Oracle语法，注意以下差异：

| 特性 | MySQL | Oracle |
|------|-------|--------|
| 自增主键 | `AUTO_INCREMENT` | `SEQUENCE + TRIGGER` 或 `IDENTITY` |
| 字符串类型 | `VARCHAR(n)` | `VARCHAR2(n)` |
| 布尔类型 | `BOOLEAN` | `NUMBER(1)` |
| 时间戳 | `DATETIME` | `DATE` 或 `TIMESTAMP` |
| 注释 | `COMMENT 'text'` | `COMMENT ON TABLE/COLUMN` |
| 引用标识符 | `` `column` `` | `"column"` |

#### Oracle语法示例

```sql
-- MySQL语法
CREATE TABLE dispatch_records (
    id INT AUTO_INCREMENT PRIMARY KEY,
    task_name VARCHAR(100) NOT NULL,
    status ENUM('计划', '执行中', '已完成', '取消'),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    COMMENT '调度记录表'
);

-- Oracle语法
CREATE SEQUENCE dispatch_records_seq START WITH 1 INCREMENT BY 1;

CREATE TABLE dispatch_records (
    id NUMBER DEFAULT dispatch_records_seq.NEXTVAL PRIMARY KEY,
    task_name VARCHAR2(100) NOT NULL,
    status VARCHAR2(20) CHECK (status IN ('计划', '执行中', '已完成', '取消')),
    created_at TIMESTAMP DEFAULT SYSTIMESTAMP
);

COMMENT ON TABLE dispatch_records IS '调度记录表';
COMMENT ON COLUMN dispatch_records.task_name IS '任务名称';
```

### 1.4 修改SQL查询（如需要）

检查以下文件中的SQL查询，确认Oracle语法兼容性：

- `src/tools/workload_statistics.py` - 工作量统计相关查询
- `src/tools/*.py` - 其他工具中的SQL查询

**需要注意的地方：**

1. **字符串函数**：
   - MySQL: `CONCAT(a, b)`
   - Oracle: `a || b` 或 `CONCAT(a, b)`

2. **日期函数**：
   - MySQL: `NOW()`, `DATE_ADD()`, `DATE_FORMAT()`
   - Oracle: `SYSTIMESTAMP`, `SYSDATE`, `TO_CHAR()`, `ADD_MONTHS()`

3. **分页**：
   - MySQL: `LIMIT 10 OFFSET 20`
   - Oracle: `OFFSET 20 ROWS FETCH NEXT 10 ROWS ONLY` (12c+)

4. **NULL处理**：
   - MySQL: `IFNULL(a, b)`
   - Oracle: `NVL(a, b)` 或 `COALESCE(a, b)`

## 2. 大模型接入

### 2.1 修改模型配置

修改 `config/agent_llm_config.json`：

```json
{
  "config": {
    "model": "YOUR_MODEL_ID",           // 修改为你的模型ID
    "temperature": 0.7,                  // 温度参数（0-1，越高越随机）
    "top_p": 0.9,                        // 核采样参数
    "max_completion_tokens": 10000,      // 最大输出token数
    "timeout": 600,                      // 超时时间（秒）
    "thinking": "disabled"               // 思考模式：enabled/disabled
  },
  "sp": "你的系统提示词...",
  "tools": ["工具列表..."]
}
```

### 2.2 获取模型ID

通过加载技能获取可用的模型ID：


- `kimi-moonshot-v1` - Kimi模型

#### 方式一：使用环境变量（推荐）

### 2.4 模型选择建议

根据你的需求选择合适的模型：

| 模型 | 适用场景 | 优势 |
|------|---------|------|
| **豆包** | 中文理解、对话交互 | 中文能力强，响应速度快 |
| **DeepSeek** | 复杂推理、代码生成 | 推理能力强，适合复杂任务 |
| **Kimi** | 长文本处理、文档分析 | 支持长上下文，适合大文档 |

**推荐配置：**
- 业务量预测：`doubao-seed-1-8-251228`
- 决策分析：`deepseek-chat`
- 文档处理：`kimi-moonshot-v1`

### 2.5 调整模型参数

根据业务需求调整参数：

```json
{
  "config": {
    "model": "doubao-seed-1-8-251228",
    "temperature": 0.3,    // 降低温度，输出更稳定（适合数据分析）
    "top_p": 0.9,
    "max_completion_tokens": 8000,  // 根据输出长度调整
    "timeout": 600,
    "thinking": "disabled"
  }
}
```

**参数说明：**
- `temperature`: 0.0-1.0，值越小输出越确定，值越大越随机
- `top_p`: 核采样，控制输出多样性
- `max_completion_tokens`: 最大输出长度
- `timeout`: 请求超时时间

```javascript
    }
}
```

## 6. 常见问题

### 6.1 Oracle连接失败

**问题**：`cx_Oracle.DatabaseError: DPI-1047: Cannot locate a 64-bit Oracle Client library`

**解决方案**：

1. 安装Oracle Instant Client：

```bash
# 下载Oracle Instant Client
# https://www.oracle.com/database/technologies/instant-client/downloads.html

# Linux
wget https://download.oracle.com/otn_software/linux/instantclient/instantclient-basic-linux.x64-21.1.0.0.0.zip
unzip instantclient-basic-linux.x64-21.1.0.0.0.zip
export LD_LIBRARY_PATH=/path/to/instantclient_21_1:$LD_LIBRARY_PATH

# macOS
brew install instantclient-basic

# Windows
# 下载并解压，添加到PATH环境变量
```

2. 或者使用 `oracledb` 包（无需Instant Client）：

```bash
pip install oracledb
```

修改 `src/storage/database/db.py`：

```python
import oracledb
oracledb.init_oracle_client()  # 如果需要
```

### 6.2 SQL语法错误

**问题**：`sqlalchemy.exc.DatabaseError: (cx_Oracle.DatabaseError) ORA-00933: SQL command not properly ended`

**解决方案**：

检查SQL语句，修改为Oracle语法：

```python
# MySQL语法
query = "SELECT * FROM table LIMIT 10"

# Oracle语法
query = "SELECT * FROM table FETCH FIRST 10 ROWS ONLY"
```

### 6.3 大模型API调用失败

**问题**：`ConnectionError: Failed to connect to model endpoint`

**解决方案**：

1. 检查API Key是否正确
2. 检查Base URL是否正确
3. 检查网络连接
4. 检查模型ID是否有效

```bash
# 测试API连接
curl -H "Authorization: Bearer YOUR_API_KEY" \
     https://your-model-endpoint.com/v1/models
```

### 6.4 前端无法加载数据

**问题**：页面显示假数据，无法加载真实数据

**解决方案**：

1. 检查后端API是否正常：

```bash
curl http://localhost:5000/api/workload_dashboard
```

2. 检查浏览器控制台是否有错误
3. 确认 `frontend/js/app.js` 中的 `loadRealTimeData()` 已启用
4. 检查数据库连接是否正常
