# 天气卡片点击事件修复说明

## 📋 问题描述

用户反馈点击天气卡片的"修改天气"按钮时报错。

## 🔍 问题原因

`showWeatherModal()` 函数没有接收 `event` 参数，导致点击事件无法正确处理。

**对比分析：**
- ✅ `showPlanWorkloadModal(event)` - 接收event参数
- ✅ `showNonPlanWorkloadModal(event)` - 接收event参数
- ❌ `showWeatherModal()` - **未接收event参数**

**HTML调用：**
```html
<div class="stat-card weather-card" onclick="showWeatherModal(event)">
    <!-- 卡片内容 -->
</div>
```

由于HTML传递了`event`参数，但函数未接收，导致函数调用失败。

## ✅ 解决方案

### 修改前（错误）
```javascript
function showWeatherModal() {
    // 创建弹窗
    // ...
}
```

### 修改后（正确）
```javascript
function showWeatherModal(event) {
    if (event) {
        event.preventDefault();
        event.stopPropagation();
    }

    // 创建弹窗
    // ...
}
```

### 修改内容

1. **添加event参数**：函数签名添加 `event` 参数
2. **处理event**：调用 `preventDefault()` 和 `stopPropagation()` 防止事件冒泡
3. **安全检查**：添加 `if (event)` 判断，避免空参数错误

## ⚠️ 注意事项

1. **event参数处理**：确保所有卡片点击事件函数都接收并处理event参数
2. **事件冒泡**：使用 `stopPropagation()` 防止事件冒泡到父元素
3. **默认行为**：使用 `preventDefault()` 阻止默认行为（如果有）

## 🌟 技术细节

### 事件处理流程
1. 用户点击天气卡片
2. HTML传递event参数给JavaScript函数
3. JavaScript函数接收event参数
4. 调用preventDefault()阻止默认行为
5. 调用stopPropagation()阻止事件冒泡
6. 显示天气详情弹窗

### 为什么需要event参数？

1. **阻止默认行为**：某些HTML元素（如<a>标签）有默认行为，需要阻止
2. **阻止事件冒泡**：防止点击事件传播到父元素，触发其他事件
3. **统一规范**：与其他卡片函数保持一致，便于维护
