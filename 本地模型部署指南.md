## 🚀 快速部署本地模型（Ollama）

### 方法1：Ollama（最简单，推荐）

#### 1. 安装 Ollama
**Windows：**
1. 访问 https://ollama.com/download
2. 下载 Windows 安装包
3. 双击安装，一路下一步
4. 验证安装：打开命令行，运行 `ollama --version`

**macOS：**
```bash
brew install ollama
```

**Linux：**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

#### 2. 启动 Ollama 服务
```bash
ollama serve
```
服务启动后，默认运行在 `http://localhost:11434`

#### 3. 拉取中文模型（推荐）
```bash
# 通义千问 7B（推荐，中文效果好，占用少）
ollama pull qwen2.5:7b

# 通义千问 14B（效果更好，需要更多资源）
ollama pull qwen2.5:14b

# Llama 3.1 8B（通用性强）
ollama pull llama3.1:8b
```

#### 4. 验证模型
```bash
# 查看已安装的模型
ollama list

# 测试模型
ollama run qwen2.5:7b "你好，请介绍一下你自己"
```

#### 5. 验证 API 服务
```bash
# 浏览器访问
http://localhost:11434/api/tags

# 或使用 curl
curl http://localhost:11434/api/tags
```

#### 6. 修改模型名称（可选）
如果拉取了其他模型，修改 `config/agent_llm_config.json` 中的 `model` 字段：
```json
{
  "config": {
    "model": "qwen2.5:14b",  // 改为你拉取的模型名称
    ...
  }
}
```

#### 7. 重启系统服务
```bash
# 停止当前服务
Ctrl + C

# 重新启动
python src/main.py
```

### 方法2：vLLM（高性能，适合GPU）

#### 1. 安装 vLLM
```bash
pip install vllm
```

#### 2. 启动服务
```bash
# 使用 Qwen 2.5 7B
python -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen2.5-7B-Instruct \
  --port 8000 \
  --dtype auto

# 使用 Llama 3.1 8B
python -m vllm.entrypoints.openai.api_server \
  --model meta-llama/Meta-Llama-3.1-8B-Instruct \
  --port 8000 \
  --dtype auto
```

#### 3. 修改 `.env` 配置
```ini
COZE_WORKLOAD_IDENTITY_API_KEY=vllm
COZE_INTEGRATION_MODEL_BASE_URL=http://localhost:8000/v1
```

#### 4. 修改 `agent_llm_config.json` 中的模型名称
```json
{
  "config": {
    "model": "Qwen/Qwen2.5-7B-Instruct",
    ...
  }
}
```

## ⚙️ 常见问题排查

### 问题1：连接失败
**错误信息：** `Can't connect to localhost:11434`

**解决方法：**
1. 检查 Ollama 服务是否启动：`ollama list`
2. 如果未启动，运行：`ollama serve`
3. 检查端口是否被占用：`netstat -ano | findstr 11434`

### 问题2：模型未找到
**错误信息：** `model not found`

**解决方法：**
1. 查看已安装模型：`ollama list`
2. 如果列表为空，拉取模型：`ollama pull qwen2.5:7b`
3. 检查 `config/agent_llm_config.json` 中的模型名称是否正确

### 问题3：响应太慢
**原因：** CPU 推理或显卡资源不足

**解决方法：**
1. 检查显卡使用情况：`nvidia-smi`
2. 如果使用 CPU，考虑升级显卡
3. 或使用更小的模型：`ollama pull qwen2.5:3b`

### 问题4：内存不足
**错误信息：** `Out of memory`

**解决方法：**
1. 使用更小的模型：`ollama pull qwen2.5:3b`
2. 减少最大 token 数：修改 `max_completion_tokens` 为 4000
3. 关闭其他占用显存的程序

## 🔧 高级配置

### 1. 调整模型参数

在 `config/agent_llm_config.json` 中：
```json
{
  "config": {
    "model": "qwen2.5:7b",
    "temperature": 0.7,        // 提高创造性
    "top_p": 0.9,             // 核采样
    "max_completion_tokens": 4000,  // 最大输出长度
    "timeout": 300,           // 超时时间（秒）
    "thinking": "disabled"    // 思考模式
  }
}
```

### 2. 启用 Ollama GPU 加速

**NVIDIA 显卡：**
```bash
# 确保 CUDA 已安装
nvidia-smi

# Ollama 会自动使用 GPU
```

**AMD 显卡：**
```bash
# 设置环境变量
set CUDA_VISIBLE_DEVICES=0

# 启动 Ollama
ollama serve
```

**Mac M 系列：**
```bash
# Ollama 会自动使用 Metal (MPS)
ollama serve
```

### 3. 多模型并行

```bash
# 拉取多个模型
ollama pull qwen2.5:7b
ollama pull llama3.1:8b

# 运行时切换模型
# 修改 config/agent_llm_config.json 中的 model 字段即可
```

## 🎓 使用建议

### 1. 模型选择
- **中文任务**：优先选择 qwen2.5 系列模型
- **通用任务**：选择 llama3.1 系列模型
- **低资源环境**：选择 qwen2.5:3b 或 qwen2.5:4b

### 2. 性能优化
- **生产环境**：使用 vLLM 获得最佳性能
- **开发环境**：使用 Ollama 方便快捷
- **GPU 不足**：使用量化模型（qwen2.5:7b-q4_K_M）

### 3. 成本控制
- **无显卡**：使用 CPU 推理（慢但免费）
- **低配置**：使用 3B-4B 模型
- **高配置**：使用 14B+ 模型
