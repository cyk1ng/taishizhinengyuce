```ini
COZE_WORKLOAD_IDENTITY_API_KEY=ollama
COZE_INTEGRATION_MODEL_BASE_URL=http://localhost:11434/v1
```

**说明：**
- `.env.local` 文件的配置会覆盖 `.env` 文件中的对应配置
- 使用 `.env.local` 的好处是：
  1. 不会影响 `.env` 文件的原始配置
  2. 方便切换不同配置（云端/本地）
  3. `.env.local` 通常不提交到版本控制

## 🚀 使用方法

### 方式一：一键启动（推荐）

#### Windows 用户
双击运行：
```
scripts\start_local_model.bat
```

#### Linux/Mac 用户
```bash
./scripts/start_local_model.sh
```

这个脚本会自动：
1. 检查 Ollama 是否安装
2. 检查并下载模型
3. 启动 Ollama 服务
4. 测试模型连接
5. 显示启动成功信息

---

### 方式二：手动部署

#### 步骤 1：安装 Ollama

**Windows：**
1. 访问 https://ollama.com/download
2. 下载并安装

**macOS：**
```bash
brew install ollama
```

**Linux：**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

---

#### 步骤 2：下载模型
```bash
ollama pull qwen2.5:7b
```

---

#### 步骤 3：启动 Ollama 服务
```bash
ollama serve
```

---

#### 步骤 4：测试模型连接
```bash
python scripts/test_local_model.py
```

---

#### 步骤 5：启动系统
```bash
python src/main.py
```

---

#### 步骤 6：访问界面
打开浏览器访问：
```
http://localhost:8000
```

## 🐛 常见问题

### Q1: 提示 "Connection refused"
**A:** Ollama 服务未启动，运行 `ollama serve`

### Q2: 提示 "model not found"
**A:** 模型未下载，运行 `ollama pull qwen2.5:7b`

### Q3: 响应速度很慢
**A:** 可能是 CPU 推理，建议使用 GPU 或更小的模型

### Q4: 内存不足
**A:** 使用更小的模型：`ollama pull qwen2.5:3b`

### Q5: 如何切换回云端模型
**A:** 参考上面的"切换回云端模型"部分
