## ⚙️ 性能优化建议

### 1. 显卡优化
**NVIDIA 显卡：**
```bash
# 确保驱动已安装
nvidia-smi

# Ollama 会自动使用 GPU
```

**AMD 显卡：**
```bash
# Windows PowerShell
$env:CUDA_VISIBLE_DEVICES=0
ollama serve

# Linux/macOS
export CUDA_VISIBLE_DEVICES=0
ollama serve
```

**Mac M 系列：**
```bash
# Ollama 会自动使用 Metal (MPS)
# 无需额外配置
```

### 2. 减少显存占用
```bash
# 使用量化模型
ollama pull qwen2.5:7b-q4_K_M

# 或减少上下文长度
# 在 config/agent_llm_config.json 中修改
{
  "config": {
    "max_completion_tokens": 4000,  // 从 8000 减少到 4000
    ...
  }
}
```

### 3. 提升推理速度
```bash
# 使用 vLLM（需要更多资源）
pip install vllm

# 启动 vLLM 服务
python -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen2.5-7B-Instruct \
  --port 8000 \
  --dtype auto

# 修改 .env
COZE_INTEGRATION_MODEL_BASE_URL=http://localhost:8000/v1
```

## 🐛 常见问题

### Q1: 提示 "Connection refused"
**A:** Ollama 服务未启动，运行 `ollama serve`

### Q2: 提示 "model not found"
**A:** 模型未下载，运行 `ollama pull qwen2.5:7b`

### Q3: 响应速度很慢
**A:** 可能是 CPU 推理，建议：
1. 检查 GPU 是否被使用：`nvidia-smi`
2. 使用更小的模型：`ollama pull qwen2.5:3b`
3. 减少输出长度：修改 `max_completion_tokens` 为 4000

### Q4: 内存不足
**A:** 建议：
1. 使用更小的模型：`ollama pull qwen2.5:3b`
2. 关闭其他占用显存的程序
3. 使用量化模型：`ollama pull qwen2.5:7b-q4_K_M`

### Q5: 如何切换回云端模型
**A:** 恢复 `.env` 文件：
```ini
COZE_WORKLOAD_IDENTITY_API_KEY=038329bc-1212-4df1-962c-6dfaf299e2c7
COZE_INTEGRATION_MODEL_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
```

恢复 `agent_llm_config.json`：
```json
{
  "config": {
    "model": "doubao-seed-1-8-251228",
    ...
  }
}
```
